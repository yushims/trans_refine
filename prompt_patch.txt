PHASE 2 - CHAIN EXECUTION:

STEP_LIBRARY (chain steps only; reference text. Do NOT execute steps not in CHAIN):

[COMBINE]
Purpose: Combine separated token fragments for identifiers/products/proper-names/entities, numbers, thousands/decimal separators, currency/finance, dates/times, web/handles, multipliers, and other tightly bound expressions when the split is meaningless (format-only).

[FORMAT]
Purpose: Fix formatting only.

[NUMERAL]
Purpose: Fix numeral formatting/normalization only.
Rules:
- Normalize unambiguous numeral expressions (including numeral-local spacing/boundary presentation such as percent/currency/version/number+unit forms) to canonical written form across languages, only when format-only (not a meaning rewrite), especially for years/dates, ordinals/cardinals, percentages, currency amounts, versions, and IDs.

- Changes must preserve pronunciation/readout intent; if pronunciation would change or is uncertain, keep original wording.
- For unambiguous spoken percentage expressions, convert to numeric percent form using `%`.
- Year policy (ASR-safe): keep 2-digit year forms unchanged by default. Expand 2-digit years to 4-digit years only with strong explicit disambiguation (e.g., nearby explicit century anchor or clearly anchored range).

[PUNCT]
Purpose: Fix punctuation/run-ons.
Rules:

- Use locked aggressiveness_level from AGGRESSIVENESS_LEVEL to choose punctuation aggressiveness (conservative/balanced/aggressive), while still following clear boundaries/clauses.

- Add missing sentence-ending punctuation before sentence-start capitalization.
- Do NOT change/add/remove punctuation at the end of the line.
- Do NOT insert punctuation adjacent to existing punctuation if it would create redundant/stacked marks.

[CASING]
Purpose: Fix wrongly cased tokens.
Rules:
- Capitalize sentence starts that are currently lowercased after sentence-ending punctuation.
# - Sentence-ending punctuation requires following sentence-start capitalization.
# - Sentence-start capitalization requires preceding sentence-ending punctuation.
# - If a casing/boundary error has already been fixed by PUNCT, do NOT revert that fix in CASING.

# - Do NOT change the casing of the first token of the line.
- Do NOT add/remove/move punctuation in this step; only change letter case.
